# Design Context

The **super-store docs site** (`docs/`, VitePress) is the project's public design surface. See
[PRODUCT.md](PRODUCT.md) for the full strategic brief and [DESIGN.md](DESIGN.md) for the visual system.

- **Register:** brand (docs is the public face of the library; design is part of the product).
- **Audience:** skeptical, evaluative TypeScript/React engineers comparing super-store to raw Yjs,
  Zustand/Jotai, or rolling their own. They read code before prose.
- **Voice:** understated, rigorous, exact. Engineer-to-engineer. No hype.

**Design principles**

1. Code is the hero, not the chrome.
2. Show the guarantee, don't sell it.
3. Quiet power — polish that's felt, not announced.
4. Restraint is identity (one violet brand, used sparingly).
5. Respect the evaluator's time (fast to scan, fast to the API).

**Avoid:** generic VitePress default, crypto/web3 maximalism, cluttered/over-decorated. Hold WCAG AA in
both themes; honor `prefers-reduced-motion`.
